# K-Means Clustering — Teens Mental Health Dataset + Real-World Survey

## Overview
This project fits a **K-Means** model on the **Teens Mental Health Dataset** (the same
dataset used in Labs 2, 3, and 4-Assignment 5) and then assigns **10 hand-crafted
teen profiles** to the learned clusters as a real-world test. The optimal $K$ is
selected with the elbow method, the fitted model and the scaler are persisted with
`joblib`, and all required visualisations are produced automatically.

## Dataset
- **Standard training set:** `dataset/Teen_Mental_Health_Dataset.csv` (1 200 rows, 13
  columns). The binary target `depression_label` is **dropped** because clustering is
  unsupervised. The remaining 12 features are used:
  `age`, `gender`, `daily_social_media_hours`, `platform_usage`, `sleep_hours`,
  `screen_time_before_sleep`, `academic_performance`, `physical_activity`,
  `social_interaction_level`, `stress_level`, `anxiety_level`, `addiction_level`.
- **Custom real-world set:** `dataset/Custom_Teen_Profiles.csv` (10 hand-crafted
  teen profiles) with identical column names and dtypes to the training set.

## Implementation
- `sklearn.cluster.KMeans` with `n_init=10` and a fixed random seed for
  reproducibility.
- `sklearn.preprocessing.StandardScaler` — fitted on the training data and then
  **re-used unchanged** on the custom data.
- Optimal $K$ selected with the elbow method (WCSS / Inertia) and confirmed with the
  silhouette score.
- Model and scaler saved to `model/220119_kmeans.pkl` and
  `model/220119_scaler.pkl` with `joblib`.
- The custom data is loaded from the local dataset folder, scaled with the same
  fitted scaler, and assigned to clusters with `model.predict`.

## Workflow
1. `!git clone` the GitHub repo (or use the local copy) so both CSVs are available.
2. Load the Teens Mental Health dataset and perform EDA.
3. Encode the three categorical columns with `LabelEncoder`.
4. Standardise all 12 features with `StandardScaler`.
5. Fit K-Means for $K = 1, \dots, 10$ and plot WCSS + silhouette.
6. Pick the optimal $K$ and fit the final K-Means.
7. Save the model and scaler with `joblib`.
8. Load the 10 custom profiles, scale them with the same scaler, and call
   `model.predict` to assign clusters.
9. Generate the elbow plot, the cluster scatter (PCA projection), the prediction
   table, and a centroid-based interpretation.

## How to Run
1. Open `220119_KMeans.ipynb` in Jupyter or Google Colab.
2. Install dependencies: `pandas`, `numpy`, `matplotlib`, `seaborn`, `scikit-learn`,
   `joblib`.
3. Run all cells. The notebook clones the GitHub repo (or uses the local copy),
   fits the model, and predicts the cluster for each custom profile — no manual
   file uploads required.

## Notes
The custom teen profiles were created with realistic, dataset-consistent values for
all 12 features. Because the column names and dtypes match the training data exactly,
the fitted `StandardScaler` can be re-applied to the custom data without any
modification.
